To study the installation of Red Hat Enterprise Linux.
Answer
STEPS:
Step 1: Boot from Installation Media
Insert the RHEL DVD/USB into the system.
Restart the computer.
Enter the BIOS/UEFI settings (usually by pressing F2, F10, or Del).
Set the DVD/USB drive as the first boot device.
Save changes and reboot.
The system will display the Red Hat Enterprise Linux boot menu.

Step 2: Start Installation
From the boot menu, select "Install Red Hat Enterprise Linux" and press Enter.
The installer will load and display the Welcome screen.

Step 3: Select Language
Choose your preferred language and keyboard layout.
Click Continue.

Step 4: Installation Summary Screen
This screen allows you to configure several options:
Date & Time – Set your region and time zone.
Keyboard – Confirm or change keyboard layout.
Language Support – Choose any additional languages if needed.
Installation Source – Usually detected automatically from the DVD/USB.
Software Selection – Choose:
Server with GUI, Minimal Install, or Workstation, depending on your use.
Installation Destination – Select your hard drive and choose Automatic partitioning (or manual if you want custom partitions).
Network & Hostname – Turn on network connection and set hostname if required.
Once all items show as “Done”, click Begin Installation.

Step 5: Set Root Password and Create User
Click Root Password and set a strong password for the root (administrator) account.
Optionally, create a regular user account.

Step 6: Installation Process
The installer will copy and install packages.
Wait until the process completes (takes about 10–30 minutes depending on system speed).

Step 7: Reboot System
After installation finishes, click Reboot.
Remove the installation media (DVD/USB) when prompted.

Step 8: First Boot Configuration
Accept the License Agreement.
Register the system (optional if you have a Red Hat subscription).
Login with your root or user account.
The RHEL desktop or command-line interface will appear.

Working with networking
Answer
# Step 1: Display the IP address of your system
ip address show
# or simply
ip addr
# → Shows network interfaces (eth0, eth1, lo) and assigned IP addresses.

# Step 2: Display the routing table (default gateway and routes)
ip route
# → Shows how packets are routed, including the default gateway.

# Step 3: View the DNS configuration file
cat /etc/resolv.conf
# → Displays the DNS servers your system is using to resolve domain names.
# Example output:
# nameserver 192.168.1.1

# Step 4: Add a new IP address to a specific network interface (temporary)
ip addr add 192.168.1.20/24 dev eth1
# → Assigns IP 192.168.1.20 to the eth1 interface.
# (This change lasts until reboot.)

# Step 5: Edit the network configuration file for eth0
vi /etc/sysconfig/network-scripts/ifcfg-Auto-eth0
# Example content:
DEVICE=eth0
BOOTPROTO=static
IPADDR=192.168.1.10
NETMASK=255.255.255.0
GATEWAY=192.168.1.1
DNS1=8.8.8.8
ONBOOT=yes
# Save and restart network:
service network restart

# Step 6: Query which DNS server is authoritative for a domain
# (You can use dig or nslookup)
dig www.sanderbanbugt.com
# or
nslookup -type=ns www.sanderbanbugt.com
# → Shows the authoritative name server (NS record) for the domain.

Securing Server with iptables
Answer
# Step 1: Display current iptables configuration
iptables -L -v

# Step 2: Set default policies to ACCEPT temporarily and flush rules
iptables -P INPUT ACCEPT
iptables -P OUTPUT ACCEPT
iptables -P FORWARD ACCEPT
iptables -F

# Step 3: Verify that policies are set and chains are flushed
iptables -L -v

# Step 4: Test network connectivity
ping 

# Step 5: Set default policy to DROP for all chains
iptables -P INPUT DROP
iptables -P OUTPUT DROP
iptables -P FORWARD DROP

# Step 6: Open loopback interface and allow essential ports (SSH, PTP)
# Allow loopback
iptables -A INPUT -i lo -j ACCEPT
iptables -A OUTPUT -o lo -j ACCEPT

# Allow SSH (port 22) and PTP (port 319/320)
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables -A OUTPUT -p tcp --sport 22 -j ACCEPT
iptables -A INPUT -p udp --dport 319:320 -j ACCEPT
iptables -A OUTPUT -p udp --sport 319:320 -j ACCEPT

# Step 7: Save current configuration
service iptables save
# or
iptables-save > /etc/sysconfig/iptables

# Step 8: Allow a specific IP to access all services
iptables -A INPUT -s  -j ACCEPT
iptables -A OUTPUT -d  -j ACCEPT

# Step 9: List rules with line numbers for verification
iptables -L -v --line-numbers

Creating Links
# Step 1: Ensure root permissions

# Step 2: Create symbolic link

cat /etc/hosts ~/symhosts ~/hardhosts
